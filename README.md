## Избранные проекты

### 0. PromptEar — Транскрибация аудио и видео с улучшением через LLM (Пет-проект)

**Самый технически сложный проект:** создание и тестирование десктопного приложения для Windows, которое преобразует аудио и видео в текст с помощью локальных нейросетей (Whisper + Qwen 2.5).

**Что тестировал:**

- **Функциональность:** загрузка 6 аудиоформатов (MP3, WAV, FLAC, OGG, M4A, AAC, WMA) и 5 видеоформатов (MP4, AVI, MOV, MKV, WEBM), валидация лимитов, обработка битых/пустых файлов.
- **LLM-логику:** проверка 3-проходного улучшения текста (очистка → грамматика/стиль → структура абзацев) на разных типах записей (диалог, монолог, шумная запись).
- **Интеграцию:** тестирование взаимодействия с Whisper (таймауты) и Ollama (падения, повторные попытки).
- **Производительность:** замер времени обработки для CPU и GPU-сборок. Выявлен рост времени для файлов > 1 ГБ.
- **Интерфейс:** проверка drag-and-drop, отображения прогресса, кнопки остановки обработки.

**Результат:**

- Найдено и задокументировано **несколько критических дефектов**, включая потерю данных при перезапуске процесса и ошибки таймаутов. Все критичные баги были исправлены до релиза.
- Составлен чек-лист на **60+ пунктов**.
- Продукт доведен до стабильного релиза (8 версий, есть сборки для CPU и CUDA, поддержка видео).

**Стек:** Postman, YouTrack, Git, ручное тестирование (Windows 10/11), Charles Proxy, FFmpeg

🔗 [Ссылка на репозиторий проекта](https://github.com/radiohd-rf/PromptEar)  
📁 [Ссылка на тестовую документацию (чек-листы, баг-репорты)](https://github.com/radiohd-rf/PromptEar/tree/master/testing)

### 1. Яндекс.Самокат — Дипломный проект (Комплексное тестирование)

**Самый масштабный проект:** тестирование веб-приложения, мобильного приложения и API сервиса заказа самокатов.

- **Веб:** Тестирование форм оформления заказа (валидация полей Имя, Телефон, Адрес, Дата доставки, Комментарий). Найдены критические баги в валидации полей.
- **Мобильное приложение (Android):** Тестирование нотификаций, поведения при отсутствии интернета, переходов по уведомлениям. Найден критический баг с падением приложения при переходе по нотификации.
- **API:** Тестирование эндпоинтов создания курьера, получения заказа, удаления курьера. Найдены ошибки валидации на сервере (принятие некорректных данных).

**Стек:** Postman, Android Studio, Redmi Note 13 Pro (Android 15), Google Sheets, YouTrack
